# Smart Parking 重构计划

## 1. Goal

将 smart-parking 从「多模块独立项目拼装」重构为「统一 monorepo」，解决：
- 子模块散落、docker-compose 无法一健启动
- 前端刚纳入，尚未与后端联动
- 项目根目录缺少统一规范

## 2. Current State

```
smart-parking/
├── RuoYi-SpringBoot3-Pro-master/   # Java 后端（RuoYi Vue3 Pro 框架）
├── RuoYi-SpringBoot3-ElementPlus-master/  # Vue3 前端
├── Parking_Ai/                      # Python AI 推理服务
├── docker-compose.yml               # 勉强可用
├── .env                            # 敏感配置
├── .cursorrules                    # Cursor 项目级规则
├── CLAUDE.md                       # 项目规范（子模块各有一份）
└── docs/                           # 计划文档
```

**主要问题：**
1. docker-compose 前端服务 healthcheck 用 curl，alpine 无 curl
2. 前端 vite.config.js 未适配 docker 代理路径（仍是 /dev-api）
3. parking-ai healthcheck 同上
4. Java Dockerfile 多阶段构建正常，但代码 src/ 目录疑似不存在（待确认）
5. 三个子模块各自有 .cursorrules + CLAUDE.md，规范碎片化
6. CI 仅覆盖 Python tests 和 Maven 部分测试，缺少前端构建
7. 根目录 .gitignore 不完整（漏了 .idea/misc.xml、*.iml）

## 3. Proposed Approach

### 3.1 统一项目规范（根目录）

保留根目录 CLAUDE.md 为唯一规范入口，删除子模块的重复 CLAUDE.md，子模块规范改为简短的 .cursorrules。

### 3.2 Monorepo 目录结构

```
smart-parking/
├── docker/                    # Docker 相关（迁移到这里）
│   ├── compose.yaml
│   ├── mysql/
│   │   └── init.sql           # 软链接到 sql/
│   └── nginx/
│       └── frontend.conf
├── apps/
│   ├── backend/               # Java 后端（RuoYi-SpringBoot3-Pro-master → apps/backend）
│   ├── frontend/              # Vue3 前端（RuoYi-SpringBoot3-ElementPlus-master → apps/frontend）
│   └── ai/                    # Python AI（Parking_Ai → apps/ai）
├── packages/                  # 共享包（未来）
├── .env.example               # 环境变量模板
├── .dockerignore
├── docker-compose.yaml       # 已在根目录
└── CLAUDE.md
```

> 目录重组**仅做软链接**，不迁移文件，保留 git 历史。改动量可控。

### 3.3 Docker 改进

- 所有 healthcheck 统一用 `wget`（alpine 默认带 busybox wget）
- MySQL init script 软链接到 `sql/smart-parking.sql`
- Java 服务内存限制 1G → 768m（实际需求）
- 前端 nginx proxy 正确路由 `/prod-api/` → 后端
- 添加 `.dockerignore` 到根目录

### 3.4 前端适配

- vite.config.js：`base: '/admin/'`、代理改为 `/prod-api`
- 前端环境变量 `VITE_API_BASE_URL` 支持 docker 注入
- 构建阶段用 node:18-alpine，生产用 nginx:alpine

### 3.5 CI 改进

- 添加前端构建 job（node + npm ci + npm run build:prod）
- 添加 docker compose 集成测试（启动所有服务 + 健康检查）

### 3.6 根目录 .gitignore 补全

补入 .idea/misc.xml、*.iml、*.ipr、*.iws、.vscode/*.json（除 settings.json）等常见 IDE 文件。

## 4. Step-by-Step Plan

> 顺序执行。每步独立，完成后 git commit。

### Step 1 — Docker 修复
**目标**：修复所有 healthcheck，验证 `docker compose up -d` 全绿。

- [ ] `docker-compose.yml` — `ruoyi-ui` healthcheck 改 `wget`
- [ ] `docker-compose.yml` — `parking-ai` healthcheck 改 `wget`，安装 curl 或改用 wget

### Step 2 — 前端适配 Docker
**目标**：前端通过 nginx 代理正确访问后端 API。

**文件**：`vite.config.js`、`Dockerfile.nginx`、`docker/nginx.conf`

### Step 3 — 添加 .dockerignore
**目标**：减少镜像体积，避免无关文件进入构建上下文。

- [ ] 创建根目录 `.dockerignore`，包含：`.git/`、`.idea/`、`*.iml`、`node_modules/`、`target/`、`*.pyc`、`__pycache__/`、`.env`、`docs/` 等

**文件**：`.dockerignore`（新建）

### Step 4 — Monorepo 目录重组（软链接）
**目标**：统一目录结构，不丢失 git 历史。

- [ ] `mkdir -p docker/mysql docker/nginx`

### Step 5 — 统一规范（删除重复 CLAUDE.md）
**目标**：消除子模块重复规范，维护单一真相来源。

- [ ] 根目录 `CLAUDE.md` 补全：加入前端规范、AI 服务规范、Docker 工作流

### Step 6 — 完善 CI
**目标**：CI 覆盖前端构建 + Docker 启动验证。

- [ ] `.github/workflows/ci.yml` — 添加 `frontend-build` job

### Step 7 — .gitignore 补全

## 6. Risks & Tradeoffs

1. **目录重组风险**：软链接在 Windows + WSL 下可能失效。**决策**：如 WSL 兼容性有问题，跳过 Step 4，仅改进内容。
2. **前端 Docker 构建**：node_modules 297MB，构建慢。**决策**：multi-stage 先 ci 再 build，利用 Docker 缓存。
3. **MySQL 初始化 SQL**：现有 sql/smart-parking.sql 依赖尚未确认。**决策**：先验证 sql 文件存在且内容正确。
4. **Java src 目录**：疑似不存在（`find` 无结果）。**决策**：先确认 Java 源码路径，可能是 IDE 模块结构问题。
