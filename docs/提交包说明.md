# 提交包说明

## 1. 两类交付内容

离线提交包用于归档和本机恢复运行，包含：

- `backend/`、`frontend/`、`ai-service/` 源码。
- `smart_parking.sql`、`scripts/` 与 `docs/`。
- `Redis/` Windows 便携运行组件。
- `ai-service/model/parking.pt` AI 推理模型权重。

Git 仓库用于保存可维护源码。仓库包含 Redis 运行组件，但不提交模型权重。

## 2. 不归档内容

以下内容不会放入离线提交包，也不会提交到 Git：

- `.env` 和其他本机凭据配置。
- `node_modules`、`target`、`dist`、缓存等生成目录。
- 日志、AI 输出图片和临时备份。
- Redis 的 `dump.rdb`、AOF、PID 和日志等运行数据。

模型权重仅进入离线提交包，不进入 Git。

## 3. 恢复运行

```powershell
cd D:\IDEA\smart-parking
Copy-Item .env.example .env
cd .\Redis
.\redis-server.exe .\redis.conf
```

随后按 [使用说明书](使用说明书.md) 配置 MySQL、导入 `smart_parking.sql` 并启动三个应用服务。

## 4. 生成提交包

```powershell
cd D:\IDEA\smart-parking
powershell.exe -ExecutionPolicy Bypass -File .\scripts\build-submission-package.ps1
```

脚本会在 `release\final-时间戳\` 生成包含模型和 Redis 的 ZIP 文件及其 SHA-256 校验文件。
