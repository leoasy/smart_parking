# 二次开发说明

## 1. 项目定位

本项目为社区车位占用检测系统，采用本地部署方式运行，由后端服务、前端管理端和 AI 推理服务组成。

项目包含开源后台管理框架的基础能力，也包含围绕车位检测场景实现的业务模块。归档或说明时应明确表述为“基于开源管理框架进行二次开发”，不要表述为从零实现完整后台框架。

## 2. 基础框架来源

后端保留了 Spring Boot 管理系统的通用能力，包括用户、角色、菜单、字典、日志、定时任务、代码生成等基础模块。为降低回归风险，部分包名和模块名仍沿用框架命名。

前端保留了 Vue3 管理端的通用能力，包括登录、权限路由、布局、菜单、表格、导入导出、富文本组件等。

这些基础能力属于开源框架和第三方库能力，源码中的许可证文件应随归档包保留。

## 3. 本项目主要实现内容

后端业务模块：

- 车位、摄像头、ROI、区域、告警、AI 事件等业务表和接口。
- 图片检测接口，负责调用 AI 服务、解析检测结果、更新车位状态。
- 状态变化事件写入逻辑，无变化时不重复新增事件。
- 智慧大屏统计接口和前端展示所需的数据聚合。

前端业务模块：

- 车位管理、摄像头管理、ROI 管理、AI 事件、告警管理等页面。
- 智慧大屏页面及车位占用、告警趋势等可视化图表。
- 本地后端代理配置和前端构建配置。

AI 推理模块：

- FastAPI 推理接口。
- YOLO 检测流程封装。
- ROI 区域读取、车位状态判断、输出图片保存。
- 检测结果回调后端并写入业务数据库。
- AI 服务测试用例和本地运行说明。

数据库与脚本：

- `smart_parking.sql` 提供本地 `smart_parking` 数据库结构和演示数据。
- `database_optimization_indexes.sql` 提供核心业务表索引优化。
- `scripts/` 提供本地依赖安装、数据库导入和启动脚本。

## 4. 归档检查说明

归档检查时建议重点说明：

- 管理系统基础能力来自开源框架，本项目保留许可证并进行二次开发。
- 本项目原创重点是车位检测业务流程、AI 推理服务、业务数据库和前端业务页面。
- Docker 配置、个人本机配置、运行日志、构建产物、缓存、模型权重不放入代码归档包。
- 模型权重如需提交，单独提供模型包，并在说明中标明用途和加载路径。

## 5. 验证命令

后端构建：

```powershell
cd D:\IDEA\smart-parking\backend
mvn.cmd -pl ruoyi-biz -am -DskipTests package
```

前端构建：

```powershell
cd D:\IDEA\smart-parking\frontend
npm.cmd run build:prod
```

```powershell
cd D:\IDEA\smart-parking\ai-service
conda run -n python39 python -m pytest -q
```
