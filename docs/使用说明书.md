# 社区车位占用检测系统使用说明书

## 1. 项目简介

本项目是一个社区车位占用检测系统。系统通过 AI 服务识别停车位占用状态，后端记录车位状态和状态变化事件，前端展示车位统计、告警趋势和业务管理页面。

## 2. 运行环境

| 组件 | 要求 |
| --- | --- |
| 操作系统 | Windows，本地 PowerShell |
| JDK | 17 或兼容版本 |
| Maven | 3.9+ |
| Node.js | 20+ |
| Python | conda 环境 `python39` |
| MySQL | 本机 3306，数据库 `smart_parking` |
| Redis | 包内 `Redis/` 或本机 Redis，6379，默认 database 1 |

## 3. 服务端口

| 服务 | 目录 | 启动端口 |
| --- | --- | --- |
| 后端 | `backend/` | 8087 |
| 前端 | `frontend/` | 5173 |
| AI 推理 | `ai-service/` | 8000 |

## 4. 初始化配置

```powershell
cd D:\IDEA\smart-parking
Copy-Item .env.example .env
```

根据本机 MySQL 和 Redis 修改 `.env`。演示环境至少需要确认：

```text
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_DATABASE=smart_parking
MYSQL_USER=Server
MYSQL_PASSWORD=你的MySQL密码
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DATABASE=1
REDIS_PASSWORD=
AI_FASTAPI_URL=http://127.0.0.1:8000
BACKEND_BASE_URL=http://127.0.0.1:8087
VITE_API_BASE_URL=http://127.0.0.1:8087
VITE_DEV_SERVER_PORT=5173
```

## 5. 导入数据库

首次运行或需要恢复演示数据时执行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\scripts\import-smart-parking-db.ps1 `
  -User Server `
  -Password "你的MySQL密码"
```

脚本会先备份当前 `smart_parking` 数据库到 `.tmp\db-backup\`，再导入根目录 `smart_parking.sql`。

## 6. 启动 Redis

离线提交包包含 Windows 便携 Redis。默认配置仅用于本机运行，未在仓库内保存口令：

```powershell
cd D:\IDEA\smart-parking\Redis
.\redis-server.exe .\redis.conf
```

若需要设置 Redis 口令，在本机副本的 `Redis\redis.conf` 中设置 `requirepass`，并将相同值写入本机 `.env` 的 `REDIS_PASSWORD`，不要把口令提交到 Git。

## 7. 安装依赖

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\scripts\install-local-deps.ps1
```

该脚本会完成三件事：

1. 安装 AI 服务 Python 依赖到 conda 环境 `python39`。
2. 安装前端 npm 依赖。
3. 打包后端 jar。

## 8. 一键启动

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\scripts\start-demo-local.ps1
```

先确保 MySQL 与 Redis 正在运行。脚本会分别打开后端、AI、前端三个 PowerShell 窗口。

## 9. 单独启动

后端：

```powershell
cd D:\IDEA\smart-parking\backend
mvn.cmd -pl ruoyi-admin -am package -DskipTests
java -jar .\ruoyi-admin\target\RuoyiSpringBoot3.jar
```

AI 服务：

```powershell
cd D:\IDEA\smart-parking\ai-service
conda run -n python39 python -m uvicorn main_api:app --host 0.0.0.0 --port 8000
```

前端：

```powershell
cd D:\IDEA\smart-parking\frontend
npm.cmd run dev -- --host 127.0.0.1 --port 5173
```

## 10. 运行流程

1. 确认 MySQL、Redis 已启动。
2. 启动后端、AI、前端。
3. 打开 `http://127.0.0.1:5173/admin/` 登录系统。
4. 进入智慧大屏或车位管理页面查看统计。
5. 调用后端检测接口，触发 AI 识别、车位状态更新和 AI 事件记录。

常用检测接口：

```text
POST http://127.0.0.1:8087/biz/parking/detect/image
```

表单参数：

| 参数 | 示例 | 说明 |
| --- | --- | --- |
| `parking_lot_id` | `roi_1` | ROI 区域编号 |
| `camera_id` | `1` | 摄像头编号 |
| `imagePath` | `D:\IDEA\smart-parking\ai-service\data\images\test.jpg` | 本机图片路径 |

AI 识别输出图片默认保存到：

```text
D:\IDEA\smart-parking\ai-service\data\outputs\
```

## 12. 归档内容

- 离线提交包包含 `Redis/` 运行组件与 `ai-service/model/parking.pt` 模型，可直接恢复运行依赖。
- Git 仓库提交源码、SQL、文档、脚本和 Redis 运行组件，不提交模型权重。
- 两种归档均不包含 `.env`、构建产物、日志、Redis 数据快照或 AI 输出图片。

## 13. 常见问题

后端 8087 无法访问：先检查后端启动日志，确认 `application.yml` 已通过根目录 `.env` 读取本机 MySQL 和 Redis 配置。

前端 `/captchaImage` 代理报错：后端未启动或端口不是 8087。先启动后端，再刷新前端页面。

AI 检测接口返回 404：确认 AI 服务启动在 8000，后端调用的是 `http://127.0.0.1:8000/parking/detect/image`。

页面出现乱码：确认数据库使用 `utf8mb4`，并重新导入 `smart_parking.sql`。
