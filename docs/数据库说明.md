# 社区车位占用检测系统数据库说明

## 1. 数据库信息

| 项 | 值 |
| --- | --- |
| 数据库名 | `smart_parking` |
| MySQL 端口 | 3306 |
| 字符集 | `utf8mb4` |
| 本地开发用户 | `Server` |

密码不要写入仓库，请放在本机 `.env` 或 PowerShell 参数中。

## 2. 导入方式

```powershell
cd D:\IDEA\smart-parking
powershell.exe -ExecutionPolicy Bypass -File .\scripts\import-smart-parking-db.ps1 `
  -User Server `
  -Password "你的MySQL密码"
```

脚本行为：

1. 检查 MySQL 连接。
2. 不存在时创建 `smart_parking` 数据库。
3. 备份当前数据库到 `.tmp\db-backup\`。
4. 导入根目录 `smart_parking.sql`。
5. 输出核心业务表行数。

## 3. 核心业务表

| 表名 | 用途 |
| --- | --- |
| `ai_event` | AI 识别后产生的车位状态变化事件 |
| `biz_alarm` | 告警记录 |
| `biz_parking_slot` | 车位基础信息和当前占用状态 |
| `biz_parking_roi` | 停车区域 ROI 信息 |
| `dev_camera` | 摄像头设备信息 |
| `biz_region` | 区域管理信息 |

## 4. 数据流

```text
本地图片 -> 后端检测接口 -> AI 推理服务 -> 后端状态更新 -> ai_event / biz_parking_slot
```

状态变化时，后端会写入 `ai_event`；无状态变化时，只返回检测结果，不新增事件记录。

## 5. 字符集检查

```sql
SELECT DEFAULT_CHARACTER_SET_NAME, DEFAULT_COLLATION_NAME
FROM information_schema.SCHEMATA
WHERE SCHEMA_NAME = 'smart_parking';
```

推荐结果：

```text
utf8mb4 / utf8mb4_0900_ai_ci
```

## 6. 演示前检查

```sql
SELECT COUNT(*) FROM biz_parking_slot;
SELECT COUNT(*) FROM ai_event;
SELECT status, COUNT(*) FROM biz_parking_slot GROUP BY status;
```

如果前端菜单或页面文字出现问号乱码，优先重新导入 `smart_parking.sql`，并确认导入时执行了 `SET NAMES utf8mb4`。
